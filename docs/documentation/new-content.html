Creating New Content

When using Brightspot, creating a new object automatically generates a user interface within the CMS, where editors can interact and publish. In this section we are going to look at how to model your objects, and the corresponding UI that is created within the CMS.

Create an object

Having installed Brightspot, you can start to create objects to appear within the CMS. To do so, create a Java class, and extend Content. Save your object within your project at src/main/java/yourGroupID. Once saved, refresh your CMS browser window and click into the Search tool. In the Create drop-down you should see your new object. Click New to open it within the content edit view. The name of your Java class will appear in the CMS as the name of the object. Below is an example Article object.

First take a look at the Java class, then the corresponding UI created within the CMS:

{% highlight java %}package com.psddev.brightspot; import com.psddev.cms.db.*; import com.psddev.cms.tool.*; import com.psddev.dari.db.*; import com.psddev.dari.util.*; import java.util.*; public class Article extends Content { private String headline; private Author author; private ReferentialText body; private List tags; private Set keywords; }{% endhighlight %}

Adding Fields

There are a range of standard UI elements that you can use. Here is a full list. Start by adding some to your newly created object. Refresh the CMS to see the new UI appear. If you want to remove any element, just remove the code within your Java class. You can always reorder the UI by changing the order of the fields in your class. The name of your field will be reflected directly in the CMS as the UI label. Below we have called out some the UI elements within our Article object:

Annotations

As well as automatically generating the user interface, directly from the java class, Brightspot provides annotations that can be used to control and assist user input and business logic. Below we have used some of the most common annotations on our Article object. You can see a full list here.

{% highlight java %} public class Article extends Content { @Required private String headline; @Indexed private Author author; private ReferentialText body; @CollectionMaximum(5) private List tags; @ToolUi.Note("Add some keywords for the article") private Set keywords; }{% endhighlight %}

Associating Objects

Once you have your first object created you can add a second. In our example we have an Author object that is associated with each Article. Associate an object by naming it as a return type in your Java class.

Every object within Brightspot has a LabelField, which is used to automatically create a label for the object. As a default, the first text string field within an object is used. The label is used when returning search results.

Our Author object is a good example of when to override the default labelfield, to help label the object more clearly. Our Author object as two fields, firstName and lastName. When an Author shows up in our search results we can only see the first name, as it is the first string text field. This can be fixed to show the two names, as the example below shows:

{% highlight java %}@LabelFields({"firstName", "lastName"}) public class Author extends Content { @Indexed @Required private String firstName; @Required private String lastName; private StorageItem file; @Embedded private Address address; private Boolean bestSeller; private Date dateOfBirth; } {% endhighlight %}

Dari Schema Tool

With several objects created you can leverage the Dari Schema Tool, which gives an instant view of your data model. Navigate to http://localhost:8080/_debug/db-schema and select one of your objects.
