Having created your own objects and published them in the CMS, the next step is to build pages on which to display your content. In this section we will build a simple page on which we will render our Article object, that we created in the previous section.

We start by adding rendering logic to our already created Article object. We add two annotations to the object, one to define the JSP that renders the Article content, and another that creates the page structure to be associated with it.

Add Renderer Paths

{% highlight java %}@Renderer.LayoutPath("/layouts/article-template.jsp") @Renderer.Path("/article-object.jsp") public class Article extends Content { private String headline; private ReferentialText body; private Author author; // Getters and Setters } {% endhighlight %}

Create JSP Files

To start, we will create two very simple files to render our page. The first is the entire page layout. As per our path specified in the annotation on our Article object, we can create a article-template.jsp file in a new directory within webapp, called "layouts". Start by importing taglibs and then create a simple structure for your page. We have added a header, which will render our headline, and a content section, which will render the rest of the object.

In the content div in the layout above we have set the value of the render as mainContent, this is defined when we create our template within the CMS, which is the final step. Now let's create the jsp file that will render the Article object. Using JSTL for our example, we call content.fieldName, where the field name is defined in our Article class. Referenced objects, such as our Author, can be accessed by name, as well as fields within them. We are rendering the Author first and last name. Create the Template



To complete the page, in the CMS under Admin > Templates & Sections create a new template, name it Article Template. See the example below:

Once you have created a template, the next step is to create a new Article and publish it. In the 'Create New' widget on the dashboard select your new Article page. Clicking in will display the content edit view, with a URL that is automatically created from your headline. Publish and access the URL to see your page with content displayed. Note, for standalone pages, such as a homepage, use the Create New drop down to build a Page, rather than a Template
